# MoPlayer Pro Windows UI Redesign

Date: 2026-06-11
Scope: Windows desktop app, NSIS installer branding, and website Windows download integration.

## What Was Improved

- Reworked the Windows app toward a premium dark glass identity with orange, amber, and copper accents.
- Replaced the previous sidebar-led navigation with a bottom glass dock inspired by MoPlayer Pro Android.
- Expanded the desktop content area now that navigation is no longer consuming a left column.
- Made movie and series grids denser so the app does not waste space with oversized cards.
- Refreshed setup artwork, header artwork, app icon generation, and release metadata.
- Connected the public website PC page to the generated Windows release metadata.
- Fixed product alias resolution so `moplayer-pc` routes to `moplayer2` activation/config logic.

## Visual System

Core colors:

- Deep black: `#020203`
- Base black: `#060608`
- Elevated charcoal: `#0e0e12`
- Graphite: `#16161c`
- Primary orange: `#ff6b2c`
- Amber: `#ff9248`
- Highlight amber: `#ffb347`
- Copper: `#e8a04a`

Design tokens live in `src/renderer/theme/tokens.css`:

- Colors, glass borders, gradients, shadows, blur levels, radius, spacing, typography, dock height, and layout spacing.
- The active accent defaults to `#ff6b2c`.
- `--dock-h`, `--dock-row-h`, and `--dock-space` reserve layout space for the bottom dock.

## Design Philosophy

- Premium media platform, not a generic desktop web shell.
- Dark cinematic base with restrained glass depth and orange glow.
- Bottom dock matches the Android Pro navigation language while staying mouse and keyboard friendly on Windows.
- Dense desktop layouts: channel rows and poster grids fit more content without oversized posters.
- QR activation and local playlist/Xtream source flows stay functional and visually consistent.

## App Changes

- `PremiumDock` was added as the main navigation surface.
- The app shell now uses a top titlebar, central screen area, and reserved bottom dock row.
- The titlebar always shows the MoPlayer Pro Windows PC brand lockup outside player mode.
- Home, Live TV, Movies, Series, Favorites, Search, Settings, License, Support, Player, and Multi-view share the same orange glass language.
- The media grid was compacted for PC so Movies/Series can show more items per row.
- Empty states, hover states, focus states, input fields, buttons, QR panel, source cards, and player controls were visually aligned.
- The player logic was not changed; the redesign is scoped to shell, presentation, and layout.

## Installer Changes

- Installer sidebar and header artwork are regenerated by `scripts/prepare-installer-art.mjs`.
- Current NSIS assets:
  - `build/installerSidebar.bmp`
  - `build/installerSidebar-preview.png`
  - `build/installerHeader.bmp`
  - `build/installerHeader-preview.png`
  - `build/license.txt`
- The setup and portable executables are generated by `npm run dist:windows`.

Technical limit:

- The current installer target is electron-builder NSIS. NSIS supports sidebar/header BMP branding and license text, but the default wizard is still a native installer flow, not a fully custom React/Electron screen. A fully bespoke installer would require a custom NSIS script/include or a separate bootstrapper.

## Website Integration

- `apps/web/src/lib/windows-release.ts` reads `public/downloads/moplayer/windows/latest-windows.json`.
- The PC landing page now displays release version, setup size, and system requirements from generated metadata.
- Download buttons route through `/api/app/download/latest?product=moplayer2&platform=windows`.
- Portable download is available when `portableFile` exists in metadata.

- `npm run verify:web`

Notes:

- Electron can print GPU/cache warnings during smoke tests in this local Windows environment; the commands exited successfully.
- The QA screenshot run can log a non-JSON warning from the remote update metadata request if the production website has not yet deployed the latest static Windows metadata. This did not fail QA.
